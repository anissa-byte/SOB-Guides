Admin Pre-Work

Before They Start

Complete ALL of these steps before scheduling the setup call. The team member cannot proceed without these.

🔒 Admin only ⏲ 15–20 min
A
Get Their GitHub Username
▶

Ask the team member to either:

  • Send you their existing GitHub username, OR
  • Go to github.com → Sign Up → Continue with Google (work account) → pick a work-friendly username → send it to you
B
Add a Seat to the GitHub Organization
▶
⚠ Lesson learned
If you skip this, the invite in Step C will fail. The org needs an available seat first.
  1. Go to github.com/orgs/School-of-Bots/people
  2. Confirm there is an available seat, or add one under Settings → Billing → Seats
  3. Then proceed to Step C
C
Invite Them to the GitHub Repo
▶

Option 1 — Slash Command in Claude Code:

Claude Code
/add-github-member [their-github-username]

Option 2 — Manual:

  1. Go to github.com/orgs/School-of-Bots/people → Invite member
  2. Enter their username → set role to Member

They will receive an email. Tell them to accept before the setup call.

D
Invite Them to the Claude Console
▶
⚠ Do not delete individual accounts
If they already have a personal Claude Console account, the org invite adds to it. They should NOT delete their individual account — doing so locks them out entirely and requires Anthropic support to recover.
  1. Go to console.anthropic.com/settings/members
  2. Click Invite → enter their work email → set role to User
  3. Tell them to accept the email invite before the setup call
E
Invite Them to the Cursor Team
▶
  1. Go to Cursor team settings (ask Natasha for admin access if needed)
  2. Invite them using their work email
  3. Have the team login info ready for the setup call
F
Generate Their API Keys
▶

Anthropic API Key

  1. Go to console.anthropic.com/settings/keys
  2. Click Create Key → name it after the team member (e.g., kat-sob-dfy)
  3. Copy the key immediately — it starts with sk-ant-. You cannot view it again.

Airtable API Key

  1. Go to airtable.com/create/tokens
  2. Click Create new token → name it after the team member
  3. Under Scopes, add: data.records:read
  4. Under Access, add the Accounts base
  5. Click Create token → copy the key

Save both keys securely — you'll paste them during the setup call.

G
Send the "You're Ready" Message
▶

Once Steps A–F are complete, send this to the team member:

Message Template
Hey! You're all set for the setup call. Before we meet:
1. Accept the GitHub invite in your email (from School-of-Bots)
2. Accept the Claude Console invite in your email (from Anthropic)
3. Download Cursor from https://www.cursor.com and drag it into Applications
4. Have your Mac password handy

That's it — we'll handle the rest together on the call. Should take about 30 minutes.

✓ You're done when

GitHub org has an available seat and invite is sent
Claude Console invite is sent
Cursor team invite is sent
Both API keys are generated and saved
"You're Ready" message is sent to the team member

Your Daily Workflow

Do this every single workday. It becomes second nature after a week.

Every Morning

1
Pull Latest & Create Your Branch
▶
  1. Open Cursor → SOB-DFY project
  2. Check the bottom-left corner — branch must say main. If not, click it and select main.
  3. Open Terminal (Ctrl + `) and run:
Terminal
git pull origin main
  1. Create today's branch: click the branch name → Create new branch → name it yourname-MM-DD-YY

During the Day

Work normally in Cursor. Run slash commands. Save with Cmd + S. No Git steps needed mid-day.

Every Evening

2
Commit, Push & Create PR
▶
  1. Click View → Source Control
  2. Hover over Changes → click the + icon to stage
  3. Write a commit message: one clear sentence
    (e.g., yourname 03-11-26 — updated client context files)
  4. Click the Commit button (checkmark icon)
  5. Click Publish Branch (first time) or Sync Changes
  6. In Terminal, run:
Terminal
gh pr create --fill
✓ You're done when
You see a URL like https://github.com/School-of-Bots/.../pull/XX

What happens overnight: Leadership reviews all pull requests before 9 AM EST. When you pull main the next morning, your approved changes are in there.

?

▶

SOB AI Handbook

How Our Systems Work

The reference guide for SOB's AI system. What our tools are, how they connect, how skills work, and how the architecture is structured.

📖 How to use this
Read this once during your first week. Come back to it whenever you need to understand how a piece of the system fits together. The AI System Rollout page in Notion is the canonical source of truth for specific skill counts, file names, and command lists.

The Big Picture

1
What Is Happening?
▶

School of Bots has built an AI agent system that automates the repeatable work across every role in our DFY service. There are dozens of skills and slash commands deployed across all roles — and more are being added as the system evolves.

Your job is evolving. Instead of doing every task from scratch, you'll use slash commands that invoke pre-built AI skills. The skills produce first-draft outputs — you review, refine, and approve. Over time, the outputs get better and your role shifts toward quality control and client strategy.

2
The Core Architecture
▶
🎯 The One Rule
Skills are process. Context is knowledge. Keep them separate.

A skill tells the AI HOW to do a task. It never changes per client.
Context tells the AI WHO it's doing it for. It changes per client.

When you run a skill for Client A, it pulls Client A's context.
When you run it for Client B, it pulls Client B's context.
The skill itself never changes. That's the whole point.

The Three Layers

LayerWhat Lives HereWho Maintains It
Skill FileThe instructions — how to do the task (universal, works for all clients)Leadership (already built)
Client ContextEverything specific to this client — voice profile, approved copy, brand notesGS + CW populate over time
AirtableLive client data — MRR, funnel status, assigned roles, flagsNatasha / Anissa maintain

You never need to build a separate skill for each client. One DM funnel copy skill works for every client we ever onboard.

You live in Layer 4. Your job is to use the commands, evaluate the outputs, and give feedback so the skills keep improving. Leadership handles Layers 1–3.

The Approved Copy Library
▶

Each client gets an Approved Copy Library — a structured document in Google Drive that stores:

  • Approved hooks (tagged by funnel and date)
  • Approved CTAs
  • Recurring transition patterns
  • Full sequences worth reusing

Who maintains it: The Copywriter adds approved copy after each funnel. The Growth Strategist confirms. Over time, the AI draws from real approved copy, generating funnels that feel consistent and require less client review.

How to Prompt AI

12
Prompting Tips
▶

Old way: Dump everything → confused output → start over.

New way: Give context in pieces → confirm understanding → next step together → iterate.

💡 The Magic Phrase
"Before you do that, come back to me with your plan first."
  • One task per conversation. Start a fresh chat for each topic.
  • Be a manager, not a suggester. Say "Write a 3-hook DM sequence" — not "Can you help me think about some DM ideas?"
  • Give context, then instruct. Share background first, confirm the AI understands, then give the task.
  • Iterate, don't restart. If the output is 70% right, tell the AI what to fix.
  • Plan in Claude.ai, build in Cursor. Brainstorm separately from your working files.

Testing & Feedback

13
How Skills Get to Production
▶
  • Run each skill at least 2 times on 2 different clients
  • Fill out the feedback form after each run (5 questions, 3 min)
  • Nothing reaches a client without leadership review
  • Production-ready = 4/5 average score, no "No" on Q1 or Q5
  • Use /feature-request for improvements or new skill ideas

Your feedback is what makes the skills better. Every issue you flag gets routed back to leadership for a fix.
